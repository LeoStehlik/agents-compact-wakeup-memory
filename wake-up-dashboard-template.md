# Wake-Up Dashboard — High Intensity State

This file is the first thing the agent should read after its identity / operating files.
It is the compact startup dashboard, not a long-term archive.

## What this file is for
- Fast startup alignment
- Current operating priorities
- Immediate blockers and watch-items
- Fresh carry-forward context from the last 24 hours
- A clear boundary on what is current vs what should be retrieved on demand

## Startup rule
Read this file early.
Do not treat it as the full memory system.
If a task needs depth, use the memory index, topic files, and search tools after reading this dashboard.

## 6. Maintenance rule
When updating this file:
- prefer current reality over completeness
- keep it short enough to read in under a minute
- remove stale focus items when they cool off
- promote only high-salience blockers, decisions, and fresh operating context
- avoid duplicating the full memory index or full task tracker
